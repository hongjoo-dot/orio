## 슬라이드 3: 프로젝트 배경 및 목표

### 배경 (AS-IS)
- 데이터가 여러 시스템에 **파편화**되어 있음
- 수동 Excel 작업으로 인한 **비효율**
- 데이터 형식이 **비표준화**되어 분석 어려움
- 실시간 현황 파악 **불가능**

### 목표 (TO-BE)
- **통합**: 모든 채널 데이터를 하나의 DB에 통합
- **자동화**: 수동 작업을 자동 파이프라인으로 대체
- **표준화**: 일관된 스키마와 데이터 거버넌스 수립
- **가시성**: 실시간 데이터 조회 및 관리 시스템 구축

## 슬라이드 7: Phase 2 - 매출 데이터 자동 수집

### DailySalesCollector

**목적**: 3개 판매 채널의 주문 데이터를 자동으로 수집하여 통합

| 채널 | 유형 | 수집 방식 | 특징 |
|------|------|----------|------|
| Cafe24 | 자사몰 | API 자동 | 10일 롤링, 출고 완료건만 |
| Sabangnet | 멀티채널 | API 자동 | 7일 롤링, 배송 확정건만 |
| ERP | 직접주문 | Excel 업로드 | 수동 선택 기간 |

**실행 스케줄**: 매일 오후 6시 (KST)

**데이터 흐름**:
API 수집 → Blob 저장 → DB MERGE → OrdersRealtime 통합

**결과**: 3개 채널 데이터가 하나의 OrdersRealtime 테이블에 통합

## 슬라이드 8: Phase 3 - 광고 데이터 자동 수집

### AdDataCollector

**목적**: Meta(Facebook) + Naver 광고 성과 데이터 자동 수집

| 플랫폼 | 수집 데이터 | 주요 지표 |
|--------|-----------|----------|
| Meta | 캠페인/광고세트/광고 성과 | Spend, ROAS, CPA, CVR, AOV |
| Naver | 검색광고 성과 | Impressions, Clicks, Conversions |

**주요 기능**:
- USD → KRW 자동 환율 변환
- Breakdown 분석 (연령/성별/플랫폼)
- Creative 정보 수집 (광고 소재)

**실행 스케줄**: 매일 오전 9시 (KST)

## 슬라이드 13: 2026년 로드맵 개요

### 비전
> **"데이터 기반 의사결정 문화 정착"**

### 4대 추진 방향

1. **가시화**
   - 부서별 맞춤형 대시보드: 단순히 전체 실적만 보는 것이 아니라, 부서별로 최적화된 지표를 제공합니다.
   - 자동 보고서 생성

2. **최적화 - 업무 자동화**
   - 데이터 정합성 자동 검증: DB 간 데이터 불일치를 실시간 모니터링하여 '숫자가 틀리지 않는 ERP' 구현.
   - 업무 프로세스 자동화(RPA): 단순 반복적인 데이터 입력이나 채널별 주문 수집 과정을 완전 자동화하여 실무자의 리소스를 전략적 업무에 집중. 

3. **지능화 - AI Agent**
   - 예측 분석 지능화: 재고 관리, 수요 예측 등 업무를 지원
   - 이상 탐지 자동화: 품절/과잉 재고 능동 알림 등

4. **확장**
   - 신규 채널/플랫폼 연동
   - 데이터 활용 범위 확대
